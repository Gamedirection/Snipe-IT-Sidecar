# Snipe-IT Sidecar

Operational playbooks for running Snipe-IT beyond inventory tracking. Covers vendor scoping, shipment tracking, security setup, asset documentation, and production handoff — gaps that Snipe-IT doesn't natively handle.

## Sections

| Section | What it covers |
|---------|---------------|
| [Vendor Scoping](vendor-scoping/README.md) | Vendor evaluation, RFx process, relationship tracking |
| [Shipment Tracking](shipment-tracking/README.md) | Receiving log, carrier integration, discrepancy handling |
| [Security Setup](security-setup/README.md) | Baseline hardening, encryption, compliance checks |
| [Documentation](documentation/README.md) | Asset documentation standards, runbooks, wiki integration |
| [Production Handoff](production-handoff/README.md) | Staging-to-production workflow, acceptance testing, go-live |

## Usage

Each section contains a README with process overview, checklists, and templates you can adapt to your organization's workflows. Import the formats into your ticketing system, internal wiki, or Snipe-IT notes fields.

### Quick Start

1. Clone this repo
2. Review each section's checklist
3. Adapt templates to your org's naming and processes
4. Reference the relevant section when onboarding new vendors, receiving shipments, hardening assets, writing docs, or handing off to production

## Why a Sidecar?

Snipe-IT excels at asset tracking, checkin/checkout, and license management. Operational processes around procurement, security, and deployment live outside its scope. This sidecar fills those gaps with reusable playbooks that complement the Snipe-IT data model.
